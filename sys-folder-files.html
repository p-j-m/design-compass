일반 규칙

  • 영문(소문자), 숫자, 언더대쉬 만을 사용
  • snake_case, kebap-case 사용

폴더

프로젝트 진행 시 사용하는 기본 폴더 구조입니다. 브랜딩과 디자인 시스템을 적용한 서비스의 경우 작업 파일에 많은 디자이너와 개발자가 접근하기 떄문에 규칙을 짜 구성원들과 공유해 혼란을 최소화합니다. 프로젝트의 성격에 따라 구성이 다를 수 있기 때문에 큰 개념을 공유하는 편입니다. 각 폴더 내부는 원본(Resources) 재료(Assets) 작업 (.sketch, .psd 등) 결과물 (_export)로 작업 순서에 따라 나뉩니다.

파일

버전 관리 프로그램을 이용할 수 있는 팀 내부 환경에서는 유니크한 이름 짓기만으로도 충분하지만, 타 조직이나 외부 업체와 프로젝트를 진행할 때는 서로 다른 규칙으로 이름이 지어진 파일을 주고 받을 경우가 많습니다. 모든 파일을 일치할 수는 없지만 파일을 단독으로 보내더라도 파일명만으로도 목적과 용도를 어느 정도 파악할 수 있어야 합니다. (예전 파일을 검색할 때마다 같은 규칙으로 파일의 이름을 표시하는게 얼마나 중요한지 알 수 있어요!)

  • prefix에는 파일 타입, 위치 등 해당 파일이 논리적으로 속해 있는 대분류를 씁니다.
  • name은 다른 요소와 구분되는 유니크한 이름을 작성합니다.
  • suffix는 구체적인 속성을 알아야하는 경우에 작성합니다.
  • prefix, name, suffix를 구분할 때는 sneak_case를 사용합니다.
  • 각 항목 내부의 단어를 구분할 필요가 있을 경우 kebap-case로 구분합니다.
